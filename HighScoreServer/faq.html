Some Common Questions & Tips

  • ScoreDoom Quickstart Guide
    (ScoreDoom only)
    Install into a separate directory.
    Grab the (optional) ScoreDoom add-on-pack (SD-ADDONPACK.WAD) to play with a ton of extra powerups & enemies
    Setup options (video/controls etc...)
    Choose a Player name in Player Options
    Choose a Player password in ScoreDoom Options (choose a separate password from your usual). You should hear a jingle noise if the password got set ok. If not, hit the console key (~) and check why it didnt get set.
    Choose Online Scoring Mode:

    • 'Off' for local hi scores (you set your own hi scores)
    • 'Regular' for competing against others on a hi score server in regular mode (saves allowed any time).
    • 'Hardcore' for competing against others on a hi score server in hardcore mode (only beginning of level auto-saves allowed).
    • 'Off' for local hi scoring only (compete against your own hi scores or against others on the same computer).
    Choose No-Infighting Mode:

    • 'Off' for local hi scores (you set your own hi scores)
    • 'Off' for normal Doom Behaviour. Monsters can fight and damage each other.
    • 'On' so that monsters neither fight nor damage each other. This increases difficulty.

    Score as many points as you can, in single player, or against others in co-op.
    For online high scoring (regular or hardcore), browse to scoredoom.com to view hi score server tables, and add a profile to the server for others to view.
  • ScoreDoomST Quickstart Guide & Hosting Tips
    (ScoreDoomST only)
    IDESD *may* first need to be set to look for scoredoomst.exe in the Options tab
    Also, for the SDST Add-On Pack, which you should install in the same directory, you should specify the directory in Option/Wads where to look for the Add-On Pack as well as other wads.

    The clients should be started with IDESD too.
    You may broadcast your server to a Master Server, or share your hostname/IP to players and have them add it to their 'Custom Server' list in IDESD Options.

    Single Player is not supported (see regular ScoreDoom for that), and the menus look just like Skulltag, but the only game mode is 'ScoreDoom Cooperative' and the only votes you can call are 'MAP', 'KICK' and 'POINTLIMIT'.

    A winner is the player with the most points after a proper finale, the end of the last map in a server map rotation, or the first player to hit the point limit.

    Please checkout the ScoreDoomST FAQ for hosting tips and other information for playing ScoreDoomST.

  • Known 'Problems'
    For regular Scoredoom check out:
    Known Problems

    ScoreDoomST is very stable, even with the Add-on Pack, but the server (and clients) can sometimes crash. The stability is no worse than the Skulltag 0.97c2 version stability that ScoreDoomST was built from. (See www.skulltag.com for more info.)

    Some ScoreDoomST Add-on Pack monsters can have sprite states that can go out of synch during play. This is actively being fixed over time, and typically is never too bad.
  • How do I get rid of the annoying *ONLINE SCORING DISABLED* purple text? (Scoredoom Online Hi Scores/Hi Score Server)
    (ScoreDoom only)
    If you just want to use local hi scoring, in ScoreDoom 'Options'->'ScoreDoom Setup', set Online Hi Scoring Mode to 'OFF'

    Scoredoom is extremely finnicky when playing with Online Hi Scoring turned on. This is to stop potential cheating. There are a quite a few reasons why you could see that text. First off, your basic gameplay options should look like this:
    'Smart Autoaim' & dmflags setup
    If you are still having problems, here are a couple of quick guides to help resolve the problem:
    Resolving Hash Code Problems
    Anti-Cheating


    When you see the *ONLINE SCORING DISABLED* text, use the '~' key to bring down the console and use 'Page Up' to view the full contents. Typically there will be a (albeit sometimes vague) reason given for why the Online Hi Scoring was disabled.
  • Different Modes, Incompatible Save Games, Purple Text!
    When playing on the hi score server, make sure to load save games only within the mode you are playing.
    i.e., if you are playing Pistol Start Mode in Non-infighting & Hardcore modes, only play with save games from that.
    These checks are to stop cheating.
  • Why do I have to restart Scoredoom whenever I change Online Hi Scoring Options, in ScoreDoom Setup?
    (ScoreDoom only)
    The zdoom engine that ScoreDoom is built on is incredibly versatile in the ways it can be configured from the console, and gameplay menus. This includes loading saved games from one scoring mode (like 'regular'), into another (like hardcore) to post a bogus score.
    To prevent cheating, ScoreDoom requires a restart to wipe the game-state of the engine completely, so all checks can be done properly to stop cheaters.
    If you like to play different modes at the same time and cant stand a restart, create seperate install directories for ScoreDoom, one for each mode.
  • ScoreDoomST Server Hosting Tips
    (ScoreDoomST only)
    Theoretically, ScoreDoomST can support up to 32 players. But typically more than 4 players in regular Doom maps in co-op *may* be too much, with regards to available ammo, health and weapons. Though you can compensate somewhat using DMFLAGS. It is recommended to use these DMFLAGS when starting a server:

    DMFLAGS
    4 - 'Weapons Remain After Pickup'
    DMFLAGS2
    2-'Drop Current Weapon Upon Death'
    64-'Double Amount of Ammo that items give you like in Skill 1 and Skill 5'
    (These DMFLAGS values are recommended for regular ScoreDoom co-op as well.)

    Playing with the Add-on Pack, with the more exotic & complicated enemies and weaponary means more entities being spawned which can eat more banwidth. If you plan to play intense wads like scythe2 with the add-on pack with more than 2 players, it may be better to have a machine solely running as the server, though your mileage may vary. I noticed when playing intense wads with the add-on pack, even with one person, that the server was much more stable if it was running on its own dedicated machine, as opposed to sharing the same machine with a player.
    Add-on Pack support will probably remain in informal 'beta' after the final release, so if running a server with it proves to be unstable, trying running a server without the pack loaded.

    See www.skulltag.com for more information on running servers in general and using IDSE (IDSD is a variant of IDSE used for launching ScoreDoomST)
  • Multiplayer Co-op: Using the Spycam to warp to other players.

    When playing Scoredoom co-op, you can view other players, via the co-op spycam (default key: 'f12'). In scoredoom 3.0, you may now also warp to the spied on player if the gameplay dmflags allows it. This is really helpful when playing wads like like Sunder and Strain which are not designed for co-op play, without turning on cheats.
    Usage: Use the 'coop spy' spycam key (default 'f12') to view the other player, then press the 'coop spy warp-to' key (default 'pause') to warp to the other player, if there is space.
    In order to use this feature, 'Allow coop spycam warp' needs to be set to 'yes' in gameplay options. For scoredoomst, this is a dmlfags setting on the server called 'allow spycam warp-to'. NOTE: If you are using IdeSD, this option is not visible under the 'dmflags2' tab, so you need to set it by typing '65536' in the 'dmlags2=' textbox. IdeSD will complain, but the flag will be set.
  • Add-on Pack: Random monsters per play session in Single Player
    (ScoreDoom only)
    When using the Add-on Pack, ScoreDoom will ensure that the same monsters will appear every time a level is played using the same wad configuration and skill level. This is to ensure that Hi Scores are meaningful.
    To have totally random monsters per play, simply set sp_random_custom_monsters=true in the <your-username>-zdoom.ini, or in sdcustom.ini (recommended).
  • Add-on Pack: Configuring the content. What is sdcustom.ini?
    (For ScoreDoom )
    sdcustom.ini is a separate custom monster config to override the default settings in '<yourusername>-zdoom.ini'.
    It is recommended to use this file when customizing Add-on Pack Content. Simply place the 'sdcustom.ini' in the same directory as gzscoredoom, or use on the command line with your own name:
    e.g. '-sdmconfig mycustom.ini'. You can even drag n' drop the ini onto the gzscoredoom.exe file.

    Editing Features:
    sdcustom.ini allows the chance (0->100) of add-on pack custom monster replacement, and their weightings to be edited easier. The same with chance (0->255) and weightings of extra health and artifact drops. Setting sp_random_custom_monsters=true will cause completely new random monsters *per play session* Now players can share custom configurations easier, as well as send the ini file with recorded demos with the add-on pack for better playback.

    NOTE: BaronChainReplacements, CyberChainReplacements & SpiderChainReplacements should always have at least one value specified. To stop boss rush mode, go into sdcustom.ini or <yourusername>-zdoom.ini and set sd_bossrush=false

    (For ScoreDoomST)
    There is no sdcustom.ini for ScoredoomST, but editing is similar. If you are using IDESD to start a server, edit the ideserver.cfg file to adjust the sdst add-on pack monster config. Else if you are launching the server without IDESD, edit the scoredoomst-.ini file config. You only need to do this for the server, since it commands what gets spawned on the clients.

    Configuring Add-On Pack Content Guide
  • Add-on Pack: Turning off Boss-Rush
    (ScoreDoom only)
    When playing with the Add-On Pack, the final levels of Ultimate Doom (E1M8, E2M8, E3M8, E4M8) will require that a series of Bosses be vanquished before the episode can be completed. Rather than the single boss(es) without the add-on pack. Some Doom2 custom levels (like in Console Doom) also support Boss Rush.

    To stop boss rush mode, go into sdcustom.ini or <yourusername>-zdoom.ini and set sd_bossrush=false
  • Add-on Pack: Default Monster Maps

    Certain maps, like MAP30 of Speed of Doom, are simply unmanageable in terms of performance when playing with the add-on pack. So you may define certain maps for a wad configuration that only use custom monsters for SpiderMasterminds, Cyberdemons & Archviles, OR no custom monsters at all, for the map within the wad configuration you are using.
    To determine the wad configuration and map you are playing, run scoredoom or scoredoomst, with the '-debugfile' option and in the 'debug.txt' look for the text 'DefaultMonsterMapsVal:'. Copy the text after it, and for Scoredoom, in the sdcustom.ini, paste the text under the '[DefaultMonsterMaps]'. In scoredoomst, this will be in your zdoom-<youusername>.ini file. For example, for Map30 of Speed of Doom, this will look like: 
    
[DefaultMonsterMaps]
|DOOM2.WAD|SD-ADDONPACK.WAD|SODFINAL.WAD|MAP30=1
    You may add as many entries as you like. By default, SpiderMasterminds, Cyberdemons and Archviles still will have custom replacements, but you may override this by setting sd_defaultmonstermaps_replace_all=true in sdcustom.ini (Scoredoom), or in zdoom-<youusername>.ini (Scoredoomst), or via the console.
    NOTE: The hi score server supports default monsters maps.
  • I like the announcer voice, but 'The Festival of Blood Continues!!!' annoys me.
    ScoreDoom's Blood-Bath announcer can be toggled on and off in the ScoreDoom Options menu. You may want to keep the announcer voice, but turn off the voice introduction everytime you start map 1, and other special maps.
    Simply set cl_bbannounce_start=false in <yourusername>-zdoom.ini
  • Add-on Pack: How come some Zdoom maps dont have Add-on Pack monsters?
    Some zdoom maps have 'specialactions' and 'behavior' (ACS) lmps defined. Scoredoom will not replace bosses for levels with a 'specialactions' lmp, so it can support the custom monster types the wad author created.
    Scoredoom will not replace *any* monsters when it encounters a 'behavior' (ACS) lmp for a level, since Add-on Pack custom monsters cannot fully support acs scripts which call Thing IDs. You can override this behaviour by setting sd_acs_level_custom_monster_override=true in <yourusername>-zdoom.ini , though this *may* make the level unfinishable.

  • How do I play multiplayer with regular Scoredoom?
    (ScoreDoom only)
    Regular Scoredoom supports multiplayer competitive co-op similar to ScoredoomST. It is best played on LAN/fast internet connections with well organized groups. This is due to unoptimized network code and it is more cumbersome to set up. This guide makes it easy:
    ScoreDoom Multiplayer HowTo

    Note: You *may* encounter 'de-synch' errors that will halt play if you play online. Use ScoredoomST for online play. ScoredoomST is much easier to setup for multiplayer play even for LAN games.
  • Zdoom HUB Support
    ScoredoomST does not support zdoom HUB levels(like in the zdoom wad 'Hell Factory').
    Regular ScoreDoom does support HUBS. For more information on HUBs and how Scoredoom deals with them, check out:
    Hubs
  • ScoreDoom Demos & Custom Add-On Pack Configs
    (ScoreDoom only)
    When you record demos with custom Add-On Pack sdcustom.ini (or whatever you named it) configurations, make sure you include your sdcustom.ini file with the demo. ScoreDoom supports drag'n drop for ini files, as well as lmp/wad/pk3 files for playing back demos easier.
